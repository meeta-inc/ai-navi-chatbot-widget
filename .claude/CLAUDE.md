# 임베디드 위젯 프로젝트 개발 지침

## 🌐 언어 사용 규칙

### 문서 및 개발 관련 (한글 사용)
- **코드 주석**: 모든 주석은 한글로 작성
- **README.md**: 한글로 작성
- **문서 파일**: 모든 `.md` 파일은 한글로 작성
- **커밋 메시지**: 한글로 작성
- **PR 설명**: 한글로 작성
- **개발자 노트**: 한글로 작성
- **CLAUDE.md**: 한글로 작성

### UI 및 사용자 대면 요소 (일본어 사용)
- **버튼 텍스트**: 일본어로 표시
- **플레이스홀더**: 일본어로 표시
- **에러 메시지**: 일본어로 표시
- **툴팁**: 일본어로 표시
- **aria-label**: 일본어로 작성
- **알림 메시지**: 일본어로 표시
- **데모 페이지 콘텐츠**: 일본어로 표시
- **모달 타이틀**: 일본어로 표시

### 예시
```javascript
// ✅ 올바른 예시
```

## 🔧 환경변수 설정

### 필수 환경변수
```env
VITE_CLIENT_ID=RS000001              # 클라이언트 ID
VITE_APP_ID=0001                     # 애플리케이션 ID
VITE_DEMO_TITLE=チャットボットテスト   # 데모 페이지 타이틀 (일본어)
VITE_CHATBOT_URL=https://...         # 챗봇 URL
```

### Amplify 배포 시 클라이언트별 설정
```bash
# 錬成会 클라이언트
VITE_CLIENT_ID=RS000001
VITE_APP_ID=0001
VITE_DEMO_TITLE=錬成会AIチャットサポート

# 名門会 클라이언트
VITE_CLIENT_ID=MM000002
VITE_APP_ID=0002
VITE_DEMO_TITLE=名門会AIアシスタント
```

## 📝 코드 컨벤션

1. **주석은 반드시 한글로 작성**
   - 코드의 의도와 로직 설명
   - TODO, FIXME 등의 마커

2. **사용자가 보는 모든 텍스트는 일본어로**
   - UI 요소 (버튼, 라벨, 메시지)
   - 에러 메시지
   - 접근성 속성 (aria-label, title)

3. **변수명과 함수명은 영어로**
   - camelCase 사용
   - 의미가 명확한 이름 사용

4. **환경변수는 VITE_ 접두사 사용**
   - 모든 환경변수는 VITE_로 시작

## ⚠️ 주의사항

1. **언어 혼용 금지**
   - 한 요소에서 한글과 일본어를 섞어 사용하지 않기
   - 일관성 있는 언어 사용

2. **번역 일관성**
   - 같은 기능의 텍스트는 항상 동일한 번역 사용
   - 용어집 참고하여 통일된 용어 사용

3. **문화적 고려사항**
   - 일본 사용자에게 자연스러운 표현 사용
   - 경어 사용 등 일본 문화에 맞는 표현
